# 三省六部 + OpenClaw 多 Agent 交互改进方案

> **分析日期**: 2026-03-14
> **当前版本**: Edict v1.0

---

## 一、当前架构痛点分析

### 1.1 数据同步层

| 问题 | 现状 | 影响 |
|------|------|------|
| **单向同步** | OpenClaw → Edict，看板操作不影响运行时 | 皇上叫停任务，Agent 可能还在跑 |
| **轮询延迟** | 15秒同步间隔 | 数据不够实时，错过关键时刻 |
| **数据割裂** | `sessions.json` vs `tasks_source.json` | 可能出现状态不一致 |

### 1.2 Agent 交互层

| 问题 | 现状 | 影响 |
|------|------|------|
| **被动上报** | Agent 需主动调用 `kanban_update.py` | 忘记上报 → 看板显示过时 |
| **无回压机制** | Agent 不感知看板状态 | 可能继续执行已取消的任务 |
| **权限硬编码** | `openclaw.json` 静态配置 | 动态调整权限需重启 Gateway |

### 1.3 可观测性层

| 问题 | 现状 | 影响 |
|------|------|------|
| **日志分散** | 各 Agent 日志独立 | 跨 Agent 调用链难以追踪 |
| **无链路追踪** | 缺少 trace_id | 问题定位困难 |
| **心跳不可靠** | 基于文件时间戳 | Agent 卡死时心跳可能仍显示正常 |

### 1.4 工作流灵活性

| 问题 | 现状 | 影响 |
|------|------|------|
| **流程固定** | 太子→中书→门下→尚书→六部 | 无法处理需要跳过门下的紧急任务 |
| **并行度受限** | 六部手动调度 | 无法自动识别并行机会 |
| **无条件分支** | 无动态路由逻辑 | 所有任务走同一流程 |

```
│   │       └─────────────┴─────────────┴─────────────┘                │    │

│   │                           ▼                                       │    │
│   │                    ┌──────────────┐                              │    │
│   │                    │ Event Hook   │ ◄── 新增：Agent 事件钩子     │    │
│   │                    │ (自动上报)   │                              │    │
│   │                    └──────────────┘                              │    │
│   └───────────────────────────────────────────────────────────────────┘    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**实现步骤**:

1. **在 OpenClaw 添加事件钩子**:
```python
# ~/.openclaw/hooks/on_state_change.py
def on_session_state_change(agent_id, session_id, old_state, new_state):
    # 自动上报到 Edict
    import requests
    requests.post('http://127.0.0.1:7891/api/internal/session-update', json={
        'agentId': agent_id,
        'sessionId': session_id,
        'oldState': old_state,
        'newState': new_state,
        'timestamp': datetime.now().isoformat()
    })
```

2. **在 Edict Backend 添加 WebSocket**:
```python
# edict/backend/app/api/websocket.py
from fastapi import WebSocket

@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    await websocket.accept()
    # 订阅 Redis channel
    async for message in redis_pubsub:
        await websocket.send_json(message)
```

3. **前端 WebSocket 连接**:
```typescript
// src/api.ts
const ws = new WebSocket('ws://127.0.0.1:7891/ws');
ws.onmessage = (event) => {
  const update = JSON.parse(event.data);
  useStore.getState().handleRealtimeUpdate(update);
};
```

**收益**:
- 延迟从 15s 降到 <100ms
- 看板操作可立即影响 Agent（通过反向控制 API）

### 2.2 【P0】任务控制反向通道

**目标**: 让看板的叫停/取消操作能真正停止运行中的 Agent

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        改进后：反向控制通道                                  │
│                                                                             │
│   看板操作                    控制指令                     Agent 响应       │
│                                                                             │
│   ┌─────────────┐         ┌────────────────┐         ┌─────────────┐      │
│   │  POST       │ ──────► │ Redis Channel  │ ──────► │ OpenClaw    │      │
│   │ /task-action│         │ task:control   │         │ Gateway     │      │
│   │ {stop}      │         │                │         │             │      │
│   └─────────────┘         └────────────────┘         └──────┬──────┘      │
│                                                               │             │
│                                                               ▼             │
│                                                        ┌─────────────┐      │
│                                                        │ Agent 进程  │      │
│                                                        │ 接收信号    │      │
│                                                        │ 优雅退出    │      │
│                                                        └─────────────┘      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**实现步骤**:

1. **Edict 发送控制指令**:
```python
# dashboard/server.py
def handle_task_action(task_id, action, reason):
    # 写入 Redis 控制通道
    redis.publish('openclaw:control', json.dumps({
        'taskId': task_id,
        'action': action,  # stop | cancel | resume
        'reason': reason,
        'timestamp': now_iso()
    }))
```

2. **OpenClaw Gateway 监听控制通道**:
```python
# 在 Gateway 中订阅控制指令
def control_listener():
    for message in redis.listen('openclaw:control'):
        cmd = json.loads(message['data'])
        if cmd['action'] == 'stop':
            # 向 Agent 发送中断信号
            agent_manager.interrupt(cmd['taskId'])
```

3. **Agent 响应中断**:
```python
# 在 Agent SOUL.md 中增加中断检查点
async def check_interrupt():
    if await redis.get('task:interrupted:{task_id}'):
        raise TaskInterrupted()
```

## 五、总结

### 核心改进方向

1. **实时性**: 轮询 → WebSocket 实时推送
2. **可控性**: 单向同步 → 双向控制
3. **可观测性**: 分散日志 → 链路追踪
4. **灵活性**: 固定流程 → 动态路由
5. **效率**: 串行派发 → 并行调度

### 投入产出比最高

| 改进 | 投入 | 产出 | ROI |
|------|------|------|-----|
| WebSocket 实时同步 | 3d | 延迟降低 99% | ⭐⭐⭐⭐⭐ |
| 反向控制通道 | 2d | 看板操作生效 | ⭐⭐⭐⭐⭐ |
| 链路追踪 | 2d | 问题定位 +50% | ⭐⭐⭐⭐ |

建议优先实施 P0 级改进，可在 1-2 周内显著提升用户体验。
