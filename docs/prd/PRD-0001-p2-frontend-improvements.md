# PRD-0001: P2 级前端改进

## Vision

为三省六部 Edict 系统提供完整的前端功能，让用户能够：
- 在看板上实时监控 Agent 状态（WebSocket 稳定连接）
- 可视化管理 Agent 调用权限
- 追踪跨 Agent 调用链路
- 监控系统性能指标

**成功标准**：
- WebSocket 断线后 5 秒内自动重连
- 权限变更实时生效（无需刷新页面）
- 追踪数据可视化展示完整调用链
- 性能指标实时更新

---

## REQ-0001-001: WebSocket 自动重连

### 动机
当前 WebSocket 连接断开后需要手动刷新页面，影响用户体验。

### 范围
- 实现指数退避重连策略
- 重连期间显示连接状态
- 重连成功后自动恢复订阅

### 非目标
- 不处理网络完全不可用的情况
- 不实现离线缓存

### 验收口径
- [ ] 断线后 5 秒内开始重连
- [ ] 最多重试 5 次，每次间隔 2^n 秒
- [ ] 重连成功后 WebSocket 连接状态为 `connected`
- [ ] 前端显示连接状态指示器

---

## REQ-0001-002: 权限管理前端界面

### 动机
后端已有动态权限矩阵 API，但前端缺乏可视化管理界面。

### 范围
- 显示当前权限矩阵
- 支持授权/撤销操作
- 显示审计日志

### 非目标
- 不实现批量操作
- 不实现权限模板

### 验收口径
- [ ] `/admin/permissions` 页面显示权限矩阵
- [ ] 点击单元格可授权/撤销
- [ ] 操作后无需刷新即可看到变更
- [ ] 审计日志显示最近 100 条记录

---

## REQ-0001-003: 追踪可视化组件

### 动机
后端已支持分布式追踪，但前端无法查看追踪数据。

### 范围
- 显示追踪列表
- 点击追踪显示调用树
- 显示 span 详情

### 非目标
- 不实现追踪搜索
- 不实现性能分析

### 验收口径
- [ ] `/traces` 页面显示最近追踪
- [ ] 点击追踪显示树形结构
- [ ] 显示每个 span 的耗时和状态

---

## REQ-0001-004: 性能监控仪表盘

### 动机
需要监控系统整体性能和健康状态。

### 范围
- 显示 Agent 活跃状态
- 显示 WebSocket 连接数
- 显示 API 响应时间

### 非目标
- 不实现历史数据查询
- 不实现告警功能

### 验收口径
- [ ] `/monitor` 页面显示性能指标
- [ ] 指标每 5 秒自动刷新
- [ ] 异常状态用红色高亮

---

## 术语表

| 术语 | 定义 |
|------|------|
| WebSocket | 实时双向通信协议 |
| Span | 追踪的基本单元，代表一个操作 |
| Trace | 多个 Span 组成的调用链 |
| 权限矩阵 | Agent 之间的调用权限表 |
